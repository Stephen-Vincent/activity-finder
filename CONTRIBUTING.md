# Contributing

This repo is currently solo-developed but PRs from collaborators are welcome.

## Reporting

- Bugs: open an issue with the `bug` label and steps to reproduce.
- Feature ideas: open an issue with the `enhancement` label.

## Submitting code

1. Fork the repo.
2. Branch from `main`: `git checkout -b feat/<short-name>` or `fix/<short-name>`.
3. Keep commits small and descriptive (Conventional Commits encouraged: `feat:`, `fix:`, `chore:`, `docs:`).
4. Run lint + typecheck before pushing.
5. Open a PR against `main` with a short description of the change and any screenshots if UI is affected.

## Development setup

See [docs/SETUP.md](./docs/SETUP.md).

## Architectural decisions

Significant decisions are recorded as ADRs under [docs/DECISIONS/](./docs/DECISIONS/). If you're proposing a change that affects architecture, add an ADR in your PR.

## Questions

Open an issue or contact the maintainer.
